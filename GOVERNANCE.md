# Project Governance

## Maintainers

Maintainers review and merge changes, manage releases, and enforce project standards.

## Decision Making

- Small changes: maintainer review and merge.
- Major API/security changes: at least two maintainer approvals recommended.

## Release Process

- Update CHANGELOG.
- Ensure `npm run check` passes.
- Tag release and publish notes.

## Security Decisions

Security-impacting decisions prioritize user safety and OWASP alignment over convenience.
